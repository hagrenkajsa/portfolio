CareSyncAI


As part of a course project in collaboration with Daresay, my team and I designed and implemented a native application for elderly home care. The purpose of the application is to facilitate communication and planning of shifts between the staff and management. One main function in the application is an AI feature that auto-generates a summary from the latest documentation about the elderly.

Task

In a team of five students with diverse expertise, we were given an open-ended task to create an optional project with minimal restrictions. The only guidance was that it should fall within the field of interaction technology and, if possible, have an innovative focus.


Project focus

The focus of the project was to use new technology and concepts rather than delivering a finished product.

Concept Idea

Our project idea emerged from a problem we observed through both personal experiences and insights from relatives: the lack of documentation and communication in elderly care. Through research, we identified that a key factor contributing to this issue is the limited time available for caregivers to document and access relevant information about the elderly.

Based on this insight, we came up with the idea to design and implement a documentation system tailored for elderly care. We chose to incorporate AI, as we believe it could enhance documentation efficiency and improve communication among caregivers. Our idea ultimately led to creating an application that allows elderly care workers to access their own and coworkers' schedules, view tasks needed for each patient, and document patient information. The app's main feature is an AI-powered function that generates summaries for specific categories of information. This summary helps caregivers in elderly care quickly access the right information, ensuring that important details aren’t lost in the large database.

Since caregivers often travel between patients and carry items in their hands, we wanted to include text-to-speech (TTS) and speech-to-text (STT) functionality to make the app more accessible. This ensures users can document and access information hands-free.

Target Group

The target group for this project consisted of people working in elderly care, including those in direct contact with patients as well as those involved in administration.

Design

When working on the design, we first created a requirement specification for all the functionality in the app. We then created a flow for the different functions.This part was challenging because of the complexity required by the system, and there were many factors we had to consider when making decisions. The key aspects in this phase were focusing on information hierarchy, accessibility, and prioritization.

Once the flow was ready, we designed wireframes, low-fidelity (lo-fi), and finally high-fidelity (hi-fi) prototypes. The design process involved several iterations after testing the prototypes with users.All designs were created in Figma, and we developed a component library to make the design process more flexible and the UI more cohesive.

AI

To generate documentation summaries, we used OpenAI’s language model. The endpoint for creating summaries takes the patient’s ID and category, then creates a summary from all the information from that specific category.



Data storage

To store the data, we used Firebase Firestore, a cloud-based NoSQL database that stores data in document format.



Speech to text & text to speech

For the TTS implementation, OpenAI's "Echo" API was used. The user's text is sent to the Node server, then to the API, which returns the audio. The server converts the audio to base64 format, which is sent back to the client and played as a sound file. For STT, we used Epo-AV to record the sound from the user and convert it to a base64. The coded audiofile was the sent to a server where OpenAI’s API Whiser that transcripted the audiofile and converted it to text.
